The Standard ML Basis Library

Change history

The following is a summary of the changes made to the SML Basis Library specification. The changes are listed with the most recent first. This list only covers actual changes or clarifications to the specification; it does not cover changes to the prose or layout.

April 12, 2004

  • Added Timer.checkCPUTimes function.

April 7, 2004

  • Clarified the definition of precision to count the hidden bit.
  • Clarified the semantics of sinh, cosh, and tanh.

September 18, 2003

Added fromNanoseconds and toNanoseconds to the Time structure.

September 15, 2003

  • Changed the type of the inputLine function in both TEXT_IO and TEXT_STREAM_IO.
  • Changed the return type of the datagram socket output functions to be unit.
  • Change the non-blocking semantics of sockets to be consistent with the PRIM_IO design (this change also avoids a nasty portablility problem).

September 12, 2003

  • Added the function realRound to the REAL signature.
  • Switched to consistent use of slices anywhere a subrange of arrays or vectors are required. The effected signatures and modules are Byte, PRIM_IO, PrimIO, Socket, and Posix.IO
  • Note that Posix.IO.readVec raises Size with a negative argument, rather than Subscript.

September 11, 2003

  • Specify all signature matching as opaque.
  • Note that setBufferMode does not permit flushing when switching between LINE_BUF and BLOCK_BUF.
  • Fix error requiring ref to live in General
  • Note that recvVec* raise Size with a negative second argument, rather than Subscript.
  • Note that implementations not providing all floating point rounding modes are non-conforming.
  • Add CharVectorSlice and CharArraySlice to TEXT.
  • Fix where type constraints on ImperativeIO and StreamIO.

September 8, 2003

Changed the names of the PACK_WORD instances to PackWord<N>Big and PackWord<N>Little.

August 26, 2003

  • Added the following optional structure instances: WordVector WordVectorSlice, WordArray, WordArraySlice, WordArray2, LargeIntVector LargeIntVectorSlice, LargeIntArray, LargeIntArraySlice, LargeIntArray2, LargeWordVector LargeWordVectorSlice, LargeWordArray, LargeWordArraySlice, LargeWordArray2, LargeRealVector LargeRealVectorSlice, LargeRealArray, LargeRealArraySlice, and LargeRealArray2.

August 19, 2003

  • Specify that Time.toSeconds, etc. round towards 0.

August 14, 2003

  • Clarification in I/O system. Changed the description of the IO.ClosedStream exception. First, we note that this exception should be used if the output stream is terminated. (If we want to distinguish the cases, we could resurrect the TerminatedStream exception.) Second, we removed the requirement that the IO.ClosedStream be reported by the writer. Since we require that output on closed streams generate IO.ClosedStream, if we want the writer to do this, the StreamIO.output function would be forced to do a write, or check somehow with the writer to see if it is closed. This is not reasonable. This still leaves open the case that the writer is closed without using the StreamIO.outstream, so the exception only occurs when the stream is flushed. But it is also possible to close the file descriptor without telling the writer.
  • Fix wrong exception in Time. The Time exception should be raised, not Overflow, if a time value cannot be represented.
  • Added possible Subscript exceptions for Windows.Reg.enumKeyEx and Windows.Reg.enumValueEx.

July 30, 2003

  • Note that ~ is overloaded for Word.word.
  • Remove deprecated Substring.all, leaving Substring.full.
  • Add WORD.toLarge, WORD.toLargeX and WORD.fromLarge to the WORD signature as synonyms for WORD.toLargeWord, WORD.toLargeWordX and WORD.fromLargeWord, and deprecate the latter. This is to be consistent with reals and ints.
  • Changed Time.fmt to raise exception on negative precisions, to be consistent with Real.fmt.
  • Note that OS.FileSys.A_EXEC should be ignored by OS.FileSys.access on systems that have no notion of execute permission.

July 30, 2003

  • Clarified the semantics of the Socket structure. Specifically, what exceptions get raised when and how closed connections are detected.
  • Removed Word8Array2 from the required structure list in the HTML version.

July 15, 2003

  • Note that Posix.IO.readVec raises Subscript if the integer argument is negative.
  • Note that top-level use function is optional.
  • Clarify meaning of StringCvt.FIX when 0 precision is specified.
  • Remove where clauses for vector and elem in TextIO as these are part of the TEXT_STREAM_IO signature.
  • Specify semantics of Error.errorName and Error.syserror.
  • Add missing where constraints to BinIO.

July 14, 2003

  • Fixed the semantics of OS.FileSys.tmpName to avoid a race condition.

July 10, 2003

  • Removed the values platformWin32Window2000 and platformWin32WindowsXP, and added platformWin32CE to Windows.Config. Dave Matthews pointed out that the first 2 are identical to platformWin32NT and can't be used to identify a platform. The last reflects the CE system documented by Microsoft.

June 26, 2003

  • Added the type Socket.sock_desc with operations sockDesc, sameDesc, and select. We also replaced the Socket.pollDesc function with ioDesc. These changes were proposed in 1998 and agreed upon, but somehow never made it into the document before now.

May 9, 2003

  • Clarified the semantics of Unix module.
  • Added note that implementations may define iodesc values not covered by OS.IO.Kind.

April 26, 2003

  • Note that the SysWord module may be provided by an implementation even when the Posix structure is not provided.

July 9, 2002

  • Removed Windows.executeWithNullStreams.

July 3, 2002

  • Changed the type of Windows.DDE.executeString so that the delay is a Time.time.
  • Changed the specification of Posix.FileSys.pathconf and Posix.FileSys.fpathconf so that a return value of false is represented by SOME 0 rather than NONE.
  • Moved the getpgrp and setpgrp functions in Posix.TTY into the substructure Posix.TTY.TC for consistency.

June 18, 2002

  • Clarified the semantics of OS.Path.toUnixPath.

December 6, 2001

  • Revised the description of the Posix.FileSys.pathconf function to be more specific.

October 29, 2001

  • Revised the Windows structure to reflect the most recent proposals by Dave Matthews and Dave Berry.
  • Removed the TerminatedStream exception from the IO module, since the operation that uses it (setPosIn) was removed previously (1998-10-05).

October 10, 2001

  • Structures specified as signature instances are matched opaquely, so all types are abstract unless explicitly specified in the signature. A note should added to the front matter stating that this is a general policy for structures matching basis library signatures.

April 11, 2001

  • More cosmetic changes of the scan type. The affected signatures are CHAR, DATE, IEEE_REAL, INTEGER, TIME, WORD.

April 3, 2001

  • Changed result type of Bool.scan to be StringCvt.reader (this is a cosmetic change).

December 15, 2000

  • Changed the OS_IO signature to make poll_desc be an eqtype.

November 20, 2000

  • Correction for slice structures: indexed operations (e.g., appi) should supply the index in the slice (not the base array).
  • Removed NetDB structure from the Basis, as there did not seem to be any way to use it.

October 31, 2000

  • Clarified the definition of OS.Process.sleep.

October 24, 2000

  • Fixed mistake in type of map and mapi in the MONO_VECTOR_SLICE signature.

October 17, 2000

  • Added the isSuffix function to the String and Substring structures.

September 20, 2000

  • Changed scan, fromString and fromDecimal in Real structure to convert extreme values to 0 or infinity, instead of raising the Overflow exception.

September 6, 2000

  • Removed references to SML90 structure.
  • Fixed Time.time to allow negative time values.
  • Extended range of offset field in Date.date to -23..23.
  • Noted that BIT_FLAGS.all may contain more set bits than those defined in a matching structure.
  • Renamed all (as used to create at slice or substring) to full to avoid conflict with the all combinator that tests a predicate over an aggregate. Maintain all in the SUBSTRING signature, but depreciate its use.
  • Fixed BIT_FLAGS.clear specification to return flags.

September 4, 2000

  • Changed Windows.Status.nocontinuableException to noncontinuableException.

July 17, 2000

  • Added the concatWith function to the STRING and SUBSTRING interfaces.

June 1, 2000

  • Added the values all, clear and intersect to BIT_FLAGS.
  • Added semantic constraints based on the correspondence with SysWord.word.
  • Added OS.Process.sleep.
  • Added option modifier to return type of Real.fromDecimal to reflect the possibility of an ill-formed argument.
  • Added Windows">Windows.executeWithNullStreams (later removed).
  • Added List.collate.
  • Added functions to ListPair module that require equal length arguments.
  • Added functions String.isSubstring and Substring.isSubstring.
  • Added update functions to VECTOR and MONO_VECTOR.
  • Fixed \x escape in Char.fromCString.
  • Specified that Unix.reap and Windows.reap are non-ephemeral.
  • Really removed Vector substructure from MONO_ARRAY2 signature (see 1998-07-21).
  • Allowed Real.scan to accept "inf" and "nan".
  • Added slice structures for arrays and vectors, and modified array and vector signatures to reflect this.
  • Added find, findi, all, exists, collate uniformly to vector, array and slice structures.

March 29, 2000

  • Added the ~ operator to the WORD signature.

Summer and Fall 1999

  • The return type of Timer.checkCPUTimer was changed to not include GC time. A new function Timer.checkGCTime was added.
  • OS.Process.status is no longer an equality type.
  • Added the type Unix.exit_status.
  • The Unix.proc type is now a type constructor. The return types of Unix.execute, Unix.executeInEnv, Unix.streamsOf, and Unix.reap were changed accordingly.
  • New functions textInstreamOf, textOutstreamOf, binInstreamOf, and binOutstreamOf were added to the Unix structure.
  • Added the function Unix.exit.
  • Added the function OS.Process.isSuccess for testing OS.Process.status values.
  • Added mkBinReader, mkTextReader, mkBinWriter, and mkTextWriter functions to the Posix.IO structure.
  • Added the function Posix.Process.fromStatus

April 28, 1999

  • Renamed POSIX_FLAGS signature to BIT_FLAGS.
  • Added the Windows structure specification.

February 26, 1999

  • Changed types of OS.FileSys.readDir and Posix.FileSys.readdir to return a string option value instead of a string.

October 5, 1998

  • Several changes to STREAM_IO signature: more specification of the relationships between inputAll, inputN, and input; changed type of inputAll and filePosIn, and deleted getPosIn, setPosIn, and in_pos, as ratified at ICFP '98.
  • In the IMPERATIVE_IO interface: deleted getPosIn, and setPosIn; added new material at end about semantics of operations and about random access.

August 28, 1998

  • Added the description of the various sockets modules.

August 26, 1998

  • Added text to specify the behaviour of Real.fromInt and Real.fromLargeInt in the case where the argument does not map exactly to a real value.

August 12, 1998

  • The functions OS.Path.mkAbsolute and OS.Path.mkRelative have been changed to take record arguments, instead of pairs.

July 21, 1998

The question about when to include substructures versus when to include types has been resolved. We are adopting the convention of favoring types over substructures, which results in the following changes:

signature STRING
Replace the Char substructure with a type specification for char.
signature SUBSTRING
Replace the String substructure with type specifications for char and string.
signature MONO_ARRAY
Replace the Vector substructure with a type specification for vector.
signature MONO_ARRAY2
Replace the Vector substructure with a type specification for vector.
In addition, there is a new structure and signature (Text : TEXT), that collect together text related structures.

May 7, 1998

There were many minor changes to the text and formatting. In addition, the following changes to the specification were made:

  • Fixed error in specification of the region type in the MONO_ARRAY2 signature.
  • Fixed error in type of CommandLine.arguments function.
  • Removed the argument to the NAN data constructor in the IEEE_REAL.float_class type, and eliminated the nan_mode type.
  • Strengthed the semantics of OS.IO.poll: the resulting list must respect the order of the argument list.
  • Changed the name of wordTo in the POSIX_FLAGS signature to fromWord.
  • Changed the specification of Char.toCString: always use three octal digits in escape sequences.

November 29, 1997

The SML Basis Library specification was re-released on the web. We do not have a list of changes prior to this release.
